Problem

지하철 내부 상황을 바로 알 수 있는 채널이 없습니다

이용객은 늘어나고 있지만, 실제로 열차 안에서 벌어지는 불편과 돌발 상황은 같은 열차를 탄 사람들끼리도 즉시 확인하기 어렵습니다.

⏱️

지연과 돌발 상황

열차가 갑자기 멈췄을 때 왜 멈췄는지, 언제 다시 출발하는지, 현재 어떤 상황인지 파악하기 어렵습니다.

내부 불편 공유 부재

취객, 소음, 냄새, 바닥 오염, 냉난방 문제처럼 열차 내부에서만 체감되는 불편은 공식 앱에서 잘 보이지 않습니다.

실시간 공감·확인 니즈

이용자들은 나만 당황한 게 아닌지, 다른 승객은 어떻게 반응하는지, 원인이 무엇인지 실시간으로 알고 싶어합니다.

Solution

승객이 직접 만드는 실시간 지하철 커뮤니티

이칸저칸은 정보 공유, 자리 정보, 커뮤니티, 민원 투표, 루틴형 가이드까지 담은 지하철 특화 서비스입니다.

정보 게시판

지연, 사고, 시위, 혼잡 등 같은 노선 이용자들이 실시간 현장 상황을 빠르게 공유합니다.

  • 노선별 현황 확인
  • 같은 열차 승객 제보
  • 실시간 체감 정보 축적

자리나눔 게시판

어디 칸이 덜 붐비는지, 어느 위치에서 앉기 유리한지 이용자 경험 기반으로 공유합니다.

  • 칸별 체감 혼잡도
  • 자리 정보 공유
  • 장시간 이용자 편의 강화

스터디 게시판

같은 노선을 자주 타는 이용자끼리 스터디, 정보 교류, 가벼운 연결이 가능합니다.

  • 통학 이용자 타깃
  • 노선 기반 네트워킹
  • 커뮤니티 체류시간 확대

자유 게시판

지하철 빌런, 노선별 분위기, 환승 팁, 공감형 이야기까지 자유롭게 나눌 수 있습니다.

  • 공감형 콘텐츠 확산
  • 자발적 참여 유도
  • SNS 바이럴 소재 확보

건의 게시판

이용자들이 필요한 게시판이나 기능을 제안하고 서비스 방향에 직접 참여할 수 있습니다.

  • 유저 참여형 개선 구조
  • 커뮤니티 니즈 반영
  • 기능 우선순위 수집

민원 투표 탭

온도, 소리, 냄새 등 칸별 불편 요소를 투표형으로 수집해 실시간 민원 모니터링 구조를 만듭니다.

  • 지하철 / 칸 선택
  • 민원 유형별 투표
  • 조치 여부 확인 가능

착석 확률 가이드

빠른 환승 중심이 아닌, 실제로 앉기 유리한 칸을 추천하는 차별 기능입니다.

  • 내릴 역 이전 역 기준 추천
  • 루틴 정보 기반 고도화
  • 향후 칸별 데이터 정교화

지하철 이용 리포트

  • 월간 이용 시간 확인
  • 지나온 역 수 집계
  • 루틴 관리 동기 부여

  • 정서적 유대감 형성
  • 반복 이용 습관화
  • 재방문 유도 요소
Core Value

이칸저칸이 주는 핵심 가치

단순 커뮤니티도 아니고, 단순 교통 앱도 아닙니다. 실시간 정보와 사용자 참여가 만나는 지하철 특화 플랫폼입니다.

⚡

실시간 정보 공유

같은 노선을 타는 이용자끼리 가장 빠른 현장 소식을 주고받습니다.

노선 기반 커뮤니티

같은 루틴을 가진 통근·통학 이용자들 사이에 연결을 만듭니다.

자리·환승 팁 축적

이용 경험이 데이터가 되고, 그 데이터가 다시 편의로 돌아옵니다.

민원 기반 개선

불편을 혼자 견디는 대신, 함께 드러내고 더 빠르게 해결할 수 있습니다.

Competitive Edge

커뮤니티와 교통 정보를 동시에

기존 서비스는 커뮤니티만 있거나 교통 정보만 있었습니다. 이칸저칸은 두 영역을 결합해 데이터가 쌓일수록 더 강해집니다.

철도 커뮤니티의 장점

사용자들끼리 체감 정보를 빠르게 나누고 공감대를 형성하는 강점이 있습니다. 하지만 정확한 교통 정보 기능은 약합니다.

️

교통 앱의 장점

이동 경로와 환승 정보는 잘 제공하지만, 같은 열차 승객끼리 소통할 수 있는 구조는 제한적입니다.

이칸저칸의 차별점

커뮤니티 + 교통 정보 + 민원 + 데이터 축적이 결합되어 시간이 지날수록 정확도와 효용이 함께 올라가는 구조입니다.

Roadmap

대학생에서 직장인, 그리고 모든 이용자로 확장

초기엔 루틴이 분명한 대학생을 중심으로 시작하고, 이후 직장인과 전 노선 이용자로 단계적으로 넓혀갑니다.

SOM · 초기 시장

지하철로 통학하는 대학생

수도권 대학가 핵심 노선을 중심으로 커뮤니티를 정착시킵니다. 실시간 상황 캡처, 공감형 콘텐츠, 대학생 커뮤니티 바이럴이 핵심입니다.

SAM · 확장 시장

지하철로 매일 통근하는 직장인

동적 환승 안내, 연착 대응, 착석 확률 고도화처럼 효율성과 예측 가능성을 높이는 방향으로 기능을 확장합니다.

TAM · 전체 시장

모든 지하철 이용자

배리어프리 안내, 엘리베이터 현황, SOS 구조까지 확장해 공익성과 실용성을 동시에 갖춘 도시 이동 인프라로 발전합니다.

Business Model

광고, 데이터, 제휴로 수익화

지역 기반 광고와 프리미엄 꾸미기, 공공데이터 활용, 교육 플랫폼 제휴까지 다층적인 수익 구조를 설계합니다.

앱 내 광고

주요 경로 기반으로 역 근처 맛집, 카페 등 상권 연계형 광고를 운영할 수 있습니다.

커스텀 아이템

프로필, 배지, 꾸미기 요소를 유료화해 참여형 서비스의 재미를 수익으로 연결합니다.

️

데이터·제휴 수익

축적된 공공 데이터를 교통·연구기관에 제공하고, 시간 맞춤형 학습 구독과도 제휴할 수 있습니다.

FAQ

발표 때 자주 나올 질문

랜딩페이지에서도 같이 보여주면 설득력이 올라가는 포인트들입니다.

정보 신뢰성은 어떻게 확보하나요?

다수 제보 중복 확인, 반응 수 기반 노출, 신고 기능, 노선별 누적 데이터로 허위 정보 확산을 줄이는 구조를 전제로 합니다.

초기 유저는 어떻게 모으나요?

대학가 핵심 노선 중심으로 시작해, 검색 유입과 공감형 SNS 콘텐츠, 커뮤니티 확산 전략을 활용합니다.

공식 기관과도 연결 가능한가요?

철도공사·교통카드 앱과의 협력 가능성을 열어두고, 초기에는 사용자 참여형 데이터 네트워크 구축에 집중합니다.

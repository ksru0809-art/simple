Problem

지하철 내부 상황을 바로 알 수 있는 채널이 없습니다

열차 안에서만 체감되는 불편과 돌발 상황은 공식 앱만으로는 파악하기 어렵습니다. 같은 열차를 탄 사람끼리 빠르게 공유할 수 있는 구조가 필요합니다.

⏱️

지연과 돌발 상황

왜 멈췄는지, 언제 다시 움직이는지 같은 핵심 정보가 늦게 전달됩니다.

🚇

칸 내부 불편

혼잡, 소음, 냄새, 냉난방 문제처럼 현장 체감 정보는 별도로 공유되지 않습니다.

💬

실시간 공감과 확인

나만 겪는 문제인지, 다른 승객도 같은 상황인지 즉시 알고 싶어하는 수요가 큽니다.

Solution

핵심 기능만 남겨 간결하게 정리했습니다

발표용 설명과 제품 소개를 동시에 잡기 위해 기능은 많아 보이되, 읽는 흐름은 무겁지 않도록 핵심 묶음만 남겼습니다.

📢

실시간 정보 게시판

지연, 사고, 혼잡 같은 현장 상황을 같은 노선 이용자들이 빠르게 공유합니다.

  • 노선별 현황 확인
  • 같은 열차 승객 제보
  • 실시간 체감 정보 축적
💺

자리 · 혼잡도 공유

어느 칸이 덜 붐비는지, 어디가 앉기 유리한지 이용자 경험 기반으로 공유합니다.

  • 칸별 체감 혼잡도
  • 자리 정보 공유
  • 착석 확률 가이드
🗳️

민원 투표와 개선

온도, 소리, 냄새 같은 칸별 불편 요소를 투표형으로 모아 빠르게 드러냅니다.

  • 지하철 / 칸 선택
  • 민원 유형별 투표
  • 불편 데이터 축적
👥

노선 기반 커뮤니티

자유 게시판, 스터디 게시판, 건의 게시판으로 사용자 간 연결과 체류 시간을 늘립니다.

  • 공감형 콘텐츠 확산
  • 통학 · 통근 루틴 연결
  • 유저 참여형 개선
🏅

루틴 관리와 배지

나의 출근 루트, 배지, 월간 리포트로 반복되는 이동을 성취 경험으로 바꿉니다.

  • 출근 루트 저장
  • 배지와 리포트
  • 재방문 유도 요소
📈

데이터가 쌓이는 서비스

이용자가 남긴 정보가 다시 다음 이용자의 편의로 돌아오는 구조를 만듭니다.

  • 루틴 정보 고도화
  • 현장 데이터 정교화
  • 시간이 갈수록 효용 증가
Core Value

커뮤니티만 있는 것도, 교통 정보만 있는 것도 아니라 둘을 실용적으로 결합했다는 점입니다.

⚡

빠른 현장성

🧭

실질적 편의

🌱

쌓일수록 강해짐

사용자 경험이 데이터가 되고, 그 데이터가 다시 서비스 경쟁력이 됩니다.

Roadmap

시장 확장 흐름도 간단하게 정리했습니다

초기 타깃은 뚜렷하게, 확장 방향은 단계적으로 보이도록 압축했습니다.

SOM

지하철 통학 대학생

대학가 핵심 노선을 중심으로 공감형 콘텐츠와 실시간 제보 문화를 먼저 만듭니다.

SAM

매일 통근하는 직장인

연착 대응, 착석 확률, 루틴 정보 고도화로 효율 중심 기능을 강화합니다.

TAM

모든 지하철 이용자

배리어프리 안내, 공공 협력, 데이터 연계까지 확장해 도시 이동 인프라로 발전합니다.

한 줄로 정리하면

이칸저칸은 지하철 안에서만 알 수 있는 현장 정보를 승객이 직접 공유하고, 그 데이터를 다시 편의 기능으로 돌려주는 실시간 커뮤니티 서비스입니다.
